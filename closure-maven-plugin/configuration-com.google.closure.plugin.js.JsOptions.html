A plexus-configurable set of options compatible with {@link CommandLineRunner}.

com.google.closure.plugin.js.JsOptions

<js>

<angularPass>BooleanGenerate $inject properties for AngularJS for functions annotated with @ngInject
<assumeFunctionWrapper>BooleanEnable additional optimizations based on the assumption that the output will be wrapped with a function wrapper. This flag is used to indicate that "global" declarations will not actually be global but instead isolated to the compilation unit. This enables additional optimizations.
<charset>StringInput and output charset for all files. By default, we accept UTF-8 as input and output US_ASCII
<checksOnly>BooleanDon't generate output. Run checks, but no optimization passes.
<compilationLevel>CompilationLevelSpecifies the compilation level to use. Options: WHITESPACE_ONLY, SIMPLE_OPTIMIZATIONS, ADVANCED_OPTIMIZATIONS
<conformanceConfig>StringA list of JS Conformance configurations in text protocol buffer format.
<dartPass>BooleanRewrite Dart Dev Compiler output to be compiler-friendly.
<debug>BooleanEnable debugging options
<define>StringOverride the value of a variable annotated @define. The format is <name>[=<val>], where <name> is the name of a @define variable and <val> is a boolean, number, or a single-quoted string that contains no single quotes. If [=<val>] is omitted, the variable is marked true
<dependencyMode>DependencyModeSpecifies how the compiler should determine the set and order of files for a compilation.

Options: NONE the compiler will include all src files in the order listed, STRICT files will be included and sorted by starting from namespaces or files listed by the --entry_point flag - files will only be included if they are referenced by a goog.require or CommonJS require or ES6 import, LOOSE same as with STRICT but files which do not goog.provide a namespace and are not modules will be automatically added as --entry_point entries. Defaults to NONE.

<entryPoint>StringA file or namespace to use as the starting point for determining which src files to include in the compilation. ES6 and CommonJS modules are specified as file paths (without the extension). Closure-library namespaces are specified with a "goog:" prefix. Example: --entry_point=goog:goog.Promise
<environment>EnvironmentDetermines the set of builtin externs to load. Options: BROWSER, CUSTOM. Defaults to BROWSER.
<exclude>StringAdd a path patterns to exclude specified as ANT-directory-scanner-style patterns like **/*.ext.
<exportLocalPropertyDefinitions>BooleanGenerates export code for local properties marked with @export
<extraAnnotationName>StringA whitelist of tag names in JSDoc. You may specify multiple
<flagFile>StringA file containing additional command-line options.
<formatting>FormattingOptionSpecifies which formatting options, if any, should be applied to the output JS. Options: PRETTY_PRINT, PRINT_INPUT_DELIMITER, SINGLE_QUOTES
<generateExports>BooleanGenerates export code for those marked with @export
<hideWarningsFor>StringIf specified, files whose path contains this string will have their warnings hidden. You may specify multiple.
<id>StringIf not supplied, autogenerated.

Compilers that may be run multiple times with different options may use the ID to disambiguate outputs.

<include>StringAdd a path patterns to include specified as ANT-directory-scanner-style patterns like **/*.ext.
<injectLibraries>BooleanAllow injecting runtime libraries.
<instrumentationFile>StringA file containing an instrumentation template.
<j2clPassMode>J2clPassModeRewrite J2CL output to be compiler-friendly if enabled (ON or AUTO). Options:OFF, ON, AUTO(default)
<jscompError>StringMake the named class of warnings an error. Must be one of the error group items. '*' adds all supported.
<jscompOff>StringTurn off the named class of warnings. Must be one of the error group items. '*' adds all supported.
<jscompWarning>StringMake the named class of warnings a normal warning. Must be one of the error group items. '*' adds all supported.
<languageIn>LanguageModeSets what language spec that input sources conform. Options: ECMASCRIPT3, ECMASCRIPT5, ECMASCRIPT5_STRICT, ECMASCRIPT6 (default), ECMASCRIPT6_STRICT, ECMASCRIPT6_TYPED (experimental)
<languageOut>LanguageModeSets what language spec the output should conform to. Options: ECMASCRIPT3 (default), ECMASCRIPT5, ECMASCRIPT5_STRICT, ECMASCRIPT6_TYPED (experimental)
<moduleOutputPathPrefix>StringPrefix for filenames of compiled JS modules. <module-name>.js will be appended to this prefix. Directories will be created as needed. Use with --module
<moduleRoot>StringPath prefixes to be removed from ES6 and CommonJS modules.
<moduleWrapper>StringAn output wrapper for a JavaScript module (optional). The format is <name>:<wrapper>. The module name must correspond with a module specified using --module. The wrapper must contain %s as the code placeholder. The %basename% placeholder can also be used to substitute the base name of the module output file.
<outputManifest>StringPrints out a list of all the files in the compilation. If --dependency_mode=STRICT or LOOSE is specified, this will not include files that got dropped because they were not required. The %outname% placeholder expands to the JS output file. If you're using modularization, using %outname% will create a manifest for each module.
<outputModuleDependencies>StringPrints out a JSON file of dependencies between modules.
<outputWrapper>StringInterpolate output into this string at the place denoted by the marker token %output%. Use marker token %output|jsstring% to do js string escaping on the output.
<outputWrapperFile>StringLoads the specified file and passes the file contents to the --output_wrapper flag, replacing the value if it exists. This is useful if you want special characters like newline in the wrapper.
<polymerPass>BooleanRewrite Polymer classes to be compiler-friendly.
<preserveTypeAnnotations>BooleanPreserves type annotations.
<printSourceAfterEachPass>BooleanWhether to iteratively print resulting JS source per pass.
<processClosurePrimitives>BooleanProcesses built-ins from the Closure library, such as goog.require(), goog.provide(), and goog.exportSymbol(). True by default.
<processCommonJsModules>BooleanProcess CommonJS modules to a concatenable form.
<processJqueryPrimitives>BooleanProcesses built-ins from the Jquery library, such as jQuery.fn and jQuery.extend()
<renamePrefixNamespace>StringSpecifies the name of an object that will be used to store all non-extern globals
<rewritePolyfills>BooleanRewrite ES6 library calls to use polyfills provided by the compiler's runtime.
<source>SourceRootBuilderSource file roots.
<sourceMapFormat>FormatThe source map format to produce. Options are V3 and DEFAULT, which are equivalent.
<sourceMapInput>StringSource map locations for input files, separated by a '|', (i.e. input-file-path|input-source-map)
<sourceMapLocationMapping>StringSource map location mapping separated by a '|' (i.e. filesystem-path|webserver-path)
<summaryDetailLevel>IntegerControls how detailed the compilation summary is. Values: 0 (never print summary), 1 (print summary only if there are errors or warnings), 2 (print summary if the 'checkTypes' diagnostic group is enabled, see --jscomp_warning), 3 (always print summary). The default level is 1
<testSource>SourceRootBuilderTest file roots.
<thirdParty>BooleanCheck source validity but do not enforce Closure style rules and conventions
<tracerMode>TracerModeShows the duration of each compiler pass and the impact to the compiled output size. Options: ALL, RAW_SIZE, TIMING_ONLY, OFF
<transformAmdModules>BooleanTransform AMD to CommonJS modules.
<translationsFile>StringSource of translated messages. Currently only supports XTB.
<translationsProject>StringScopes all translations to the specified project.When specified, we will use different message ids so that messages in different projects can have different translations.
<useNewTypeInference>BooleanChecks for type errors using the new type inference algorithm.
<useTypesForOptimization>BooleanEnable or disable the optimizations based on available type information. Inaccurate type annotations may result in incorrect results.
<version>BooleanPrints the compiler version to stdout and exit.
<warningLevel>WarningLevelSpecifies the warning level to use. Options: QUIET, DEFAULT, VERBOSE
<warningsWhitelistFile>StringA file containing warnings to suppress. Each line should be of the form <file-name>:<line-number>? <warning-description>
