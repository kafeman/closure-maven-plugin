JavaScript-backend related options for the Soy compiler.

com.google.closure.plugin.soy.Js

<js>

<bidiGlobalDir>TextDirectionThe bidi global directionality as a static value, If unspecified, and useGoogIsRtlForBidiGlobalDir is false, the bidi global directionality will actually be inferred from the message bundle locale. This must not be the case when shouldGenerateGoogMsgDefs is true, but is the recommended mode of operation otherwise.
<googMsgsAreExternal>BooleanWhether the Closure Library messages are external, i.e. "MSG_EXTERNAL_[soyGeneratedMsgId]".
<id>StringIf not supplied, autogenerated.

Compilers that may be run multiple times with different options may use the ID to disambiguate outputs.

<shouldDeclareTopLevelNamespaces>BooleanWhether we should generate code to declare the top level namespace.
<shouldGenerateGoogModules>BooleanWhether we should generate code to declare goog.modules.
<shouldGenerateGoogMsgDefs>BooleanWhether we should generate Closure Library message definitions (i.e. goog.getMsg).
<shouldGenerateJsdoc>BooleanWhether we should generate JSDoc with type info for the Closure Compiler.
<shouldProvideBothSoyNamespacesAndJsFunctions>BooleanWhether we should generate code to provide both Soy namespaces and JS functions.
<shouldProvideRequireJsFunctions>BooleanWhether we should generate code to provide/require template JS functions.
<shouldProvideRequireSoyNamespaces>BooleanWhether we should generate code to provide/require Soy namespaces.
<useGoogIsRtlForBidiGlobalDir>BooleanWhether to determine the bidi global direction at template runtime by evaluating goog.i18n.bidi.IS_RTL. May only be true when both shouldGenerateGoogMsgDefs and either shouldProvideRequireSoyNamespaces or shouldProvideRequireJsFunctions is true.
</>
